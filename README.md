Go implementation of the [Scout-and-Wave protocol](https://github.com/blackwell-systems/scout-and-wave) for parallel agent coordination.

The protocol repo defines the SAW specification. This repo is the Go CLI implementation — you do not need to clone or install the protocol repo separately.

## Prerequisites

- **Go 1.25 or later** — required by `go.mod`; `go install` will fail on older versions
- **git** — SAW creates and merges git worktrees during wave execution
- **`claude` CLI** *(optional)* — required only when using `--backend cli` (Claude Max plan, no API key needed); install from [claude.ai/code](https://claude.ai/code)
- **`ANTHROPIC_API_KEY`** *(optional)* — required only when using `--backend api`; the default `auto` mode uses whichever is available

## Overview

SAW coordinates multiple AI agents working in parallel on non-overlapping parts of a codebase. Each agent gets an isolated git worktree (an isolated git working directory per agent); the orchestrator merges, verifies, and sequences waves (groups of parallel agents working on non-overlapping files) automatically.

## Concepts

| Term | Definition |
|------|-----------|
| **IMPL doc** | A Markdown planning document generated by `saw scout` that drives all subsequent commands, stored in `docs/IMPL/`. It lists waves, agents, file ownership, and interface contracts. |
| **wave** | A group of parallel agents that work simultaneously on non-overlapping files. Wave N+1 does not start until Wave N is merged and verified. |
| **scaffold** | Shared type stubs and interface files created before agents run, so parallel agents can compile against agreed-upon signatures. |
| **worktree** | An isolated git working directory created for each agent (`saw/waveN-agent-X` branch). Agents commit to their worktree; the orchestrator merges them. |
| **I1–I6** | The six protocol invariants that SAW enforces: disjoint file ownership (I1), interface contracts precede execution (I2), sequential waves (I3), IMPL doc as single source of truth (I4), agents commit before reporting (I5), and role separation (I6). |

## Quickstart

```bash
# Step 1 — Analyze your codebase and generate an IMPL doc.
# This calls the Scout agent, which reads your repo and produces
# docs/IMPL/IMPL-<slug>.md — the plan that drives all subsequent steps.
saw scout --feature "add caching layer"

# Step 2 — Review the generated IMPL doc.
# Open docs/IMPL/IMPL-add-caching-layer.md and verify the agent assignments,
# file ownership, and interface contracts look correct.
# Optionally use the web UI for an interactive review:
saw serve   # then open http://localhost:7432

# Step 3 — Run all waves automatically.
# SAW spawns parallel agents per wave, merges each wave, runs tests,
# and proceeds to the next wave without prompts.
saw wave --impl docs/IMPL/IMPL-add-caching-layer.md --auto
```

See `examples/sample-impl.md` for a sample IMPL doc showing the format that `saw scout` produces.

## Installation

```bash
go install github.com/blackwell-systems/scout-and-wave-go/cmd/saw@latest
```

## Usage

```bash
# Analyze a codebase and produce an IMPL doc
# (an IMPL doc is a Markdown planning document generated by `saw scout`
#  that drives all subsequent commands, stored in docs/IMPL/)
saw scout --feature "add OAuth support"

# Create shared type scaffold files from an IMPL doc
saw scaffold --impl docs/IMPL/IMPL-oauth.md

# Execute all waves automatically (no prompts)
saw wave --impl docs/IMPL/IMPL-oauth.md --auto

# Execute from a specific wave number
saw wave --impl docs/IMPL/IMPL-oauth.md --wave 2 --auto

# Merge a single wave manually (recovery)
saw merge --impl docs/IMPL/IMPL-oauth.md --wave 1

# Check wave/agent status
saw status --impl docs/IMPL/IMPL-oauth.md
saw status --impl docs/IMPL/IMPL-oauth.md --json
saw status --impl docs/IMPL/IMPL-oauth.md --missing

# Open the web UI (review IMPL docs + live wave dashboard)
saw serve

# Print version
saw --version
```

### Backend selection

`saw scout` and `saw scaffold` support a `--backend` flag and `SAW_BACKEND` environment variable:

| Value | Behavior |
|-------|----------|
| `api` | Calls the Anthropic API directly. Requires `ANTHROPIC_API_KEY`. |
| `cli` | Shells out to `claude --print`. Works with Claude Max plan — no API key needed. |
| `auto` | Uses `api` when `ANTHROPIC_API_KEY` is set, `cli` otherwise. **Default.** |

```bash
# Explicit flag
saw scout --feature "add OAuth support" --backend cli

# Persistent default via env var (flag takes precedence)
export SAW_BACKEND=cli
saw scout --feature "add OAuth support"
```

### Web UI (`saw serve`)

`saw serve` starts a local HTTP server and opens the browser automatically on macOS and Linux.

- **IMPL picker** — select any IMPL doc from the home screen; no manual slug entry required
- **Review screen** — suitability badge, file ownership table, wave diagram, interface contracts, approve/reject buttons
- **Wave dashboard** — live per-wave progress bars and agent cards (status, files, errors) streamed over SSE
- **Dark mode** — toggle persisted to `localStorage`

Flags: `--addr` (default `localhost:7432`), `--impl-dir`, `--repo`, `--no-browser`

## Building from source

The web UI (`web/dist/`) is embedded into the Go binary via `go:embed`. Running `go build ./cmd/saw/` alone will fail if `web/dist/` is absent. Use `make build` instead, which builds the web UI first:

```bash
# Prerequisites: Node.js (for the web build step)
make build
```

If you only want to build without the web UI (e.g., for backend-only development), run `cd web && npm install && npm run build` once to populate `web/dist/`, then `go build ./cmd/saw/` as normal.

## Architecture

```
cmd/saw/           # CLI entry point (wave, status, scout, scaffold, merge, serve)
pkg/
├── orchestrator/  # 10-state machine + wave coordination + merge procedure + SSE events
├── protocol/      # IMPL doc parser + completion report extraction
├── worktree/      # Git worktree create/remove/cleanup
├── agent/
│   ├── backend/   # Backend interface (api + cli implementations)
│   └── ...        # Runner + completion polling
└── api/           # HTTP server, SSE broker, REST endpoints
internal/
└── git/           # Git CLI wrappers (worktree, merge, diff, rev-parse)
web/               # React + TypeScript + Tailwind (baked into binary via go:embed)
```

## Protocol Compliance

Implements [SAW Protocol v0.8.0](https://github.com/blackwell-systems/scout-and-wave/tree/main/protocol):

| Invariant | Description |
|-----------|-------------|
| I1 | Disjoint file ownership enforced pre-merge |
| I2 | Interface contracts precede parallel execution |
| I3 | Wave N+1 blocked until Wave N merged + verified |
| I4 | IMPL doc is single source of truth |
| I5 | Agents commit before reporting (enforced by merge trip wire) |
| I6 | Role separation: orchestrator does not do agent work |

**10-state machine:**

```
ScoutPending → Reviewed → ScaffoldPending → WavePending → WaveExecuting
                                                 ↑              ↓
                                            WaveVerified ← WaveMerging
                                                 ↓
                                             Complete
```

Terminal states: `NotSuitable`, `Complete`. Recovery state: `Blocked`.
